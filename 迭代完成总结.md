### ✅ 3. 关键问题修复

#### 问题 A: 固定功能管道不可用
```
症状: glMatrixMode, glVertexPointer 等调用报错 "invalid operation (1282)"
原因: OpenGL 4.1 Core Profile 不支持
解决: CPU 端计算 MVP 矩阵，通过着色器使用
```

#### 问题 B: PyOpenGL 上下文检查失败
```
症状: "Attempt to retrieve context when no valid context"
原因: PyOpenGL 的上下文检查在 GTK GLArea 回调中不工作
解决: 切换到 ModernGL (使用 libGL 直接调用)
```

#### 问题 C: 投影矩阵错误
```
症状: 所有点都在裁剪空间外 (x: -30 ~ 30)
原因: 投影矩阵元素位置颠倒
修复: [2,2]=-(zf+zn)/(zn-zf), [2,3]=-2*zf*zn/(zn-zf), [3,2]=-1
结果: 坐标范围正确 (x: -1 ~ 1)
```

#### 问题 D: ModernGL VAO 创建失败
```
症状: "missing data or reserve"
原因: 空缓冲区无法创建 VAO
解决: 将 VAO 创建延迟到 upload_gpu_buffers (有数据时)
```

#### 问题 E: 导入冲突
```
症状: ModuleNotFoundError: gtk_viewer_moderngl
原因: 残留的空文件被导入
解决: 删除 gtk_viewer_moderngl.py, 修正导入
```

## 技术亮点

### 1. CPU 端 MVP 计算
```python
# 标准的透视投影矩阵
def create_projection_matrix(fovy, aspect, znear, zfar):
    f = 1.0 / tan(fovy/2)
    P[0,0] = f/aspect
    P[1,1] = f
    P[2,2] = -(zfar+znear)/(zfar-znear)
    P[2,3] = -2*zfar*znear/(zfar-znear)
    P[3,2] = -1
    return P

# 正确的视图矩阵应用顺序
V = T(0,0,-dist) @ Ry(yaw) @ Rx(pitch) @ T(-center)

# 裁剪空间坐标计算
clip_pos = positions @ (P @ V).T
ndc_pos = clip_pos[:,:3] / clip_pos[:,3:4]  # 透视除法
```

### 2. ModernGL 集成
```python
# 创建程序和 VAO
mgl_ctx = moderngl.create_context()  # 使用 GTK 的上下文
mgl_prog = mgl_ctx.program(vertex_shader=vert_src, fragment_shader=frag_src)
mgl_vao = mgl_ctx.vertex_array(mgl_prog, [...])  # 延迟创建

# 渲染
transformed_pos = get_transformed_positions()  # CPU 计算
mgl_pos_vbo.write(transformed_pos.tobytes())   # 上传
mgl_vao.render(mode=moderngl.POINTS)           # 渲染
```

### 3. LOD 系统
```
# 三级 LOD 架构
├─ full    : 使用所有点 (静止视图)
├─ medium  : 50% 采样 (缩放时)
└─ preview : 10% 采样 (拖动交互时, 保持 60+ FPS)

# 交互模式自动切换
on_button_press (右键)   → interactive=True  → 使用 preview LOD
on_button_release        → interactive=False → 使用 full/medium LOD
```
