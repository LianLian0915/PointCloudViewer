## ❌ 未完成 - 关键问题

### PyOpenGL Context检测问题

**症状**：顶点属性指针设置失败
```
OpenGL.error.Error: Attempt to retrieve context when no valid context
- 在 glBindVertexArray()
- 在 glEnableVertexAttribArray()  
- 在 glVertexAttribPointer()
```

**奇怪的是**：
- ✅ glGetString(GL_VERSION) 成功
- ✅ glGetUniformLocation() 成功
- ✅ glUniformMatrix4fv() 成功
- ✅ glDrawArrays() 成功
- ❌ glBindVertexArray() 失败
- ❌ glEnableVertexAttribArray() 失败
- ❌ glVertexAttribPointer() 失败

**根本原因分析**：
- GTK GLArea 提供 Core Profile context
- PyOpenGL的contextdata模块过于严格地检查context可用性
- 某些GL函数通过了检查，某些没有
- Context实际上是活跃的（glClear, glViewport工作）

**尝试过的解决方案**：
1. ❌ 在initializeGL中设置VAO - 失败
2. ❌ 在on_render中延迟设置 - 失败
3. ❌ 禁用PyOpenGL ERROR_CHECKING - 失败
4. ❌ 禁用PyOpenGL ALLOW_NUMPY_FUNCTIONS - 失败
5. ❌ 使用ctypes传递参数 - 失败
6. ❌ 调用make_current() - 失败
7. ❌ 使用glBindAttribLocation预绑定 - 失败

## 🔍 技术细节

### 工作的代码路径
```
on_render() 
  → initializeGL()
    ✅ glGenBuffers
    ✅ glGenVertexArrays
    ✅ compileShader / compileProgram
    ✅ glGetUniformLocation
  → paintGL()
    ✅ glClear
    ✅ glViewport
    ✅ glBufferData
    ✅ glUseProgram
    ✅ glUniformMatrix4fv
    ✅ glDrawArrays
    ❌ glBindVertexArray (在paintGL中)
    ❌ glEnableVertexAttribArray (在paintGL中)
    ❌ glVertexAttribPointer (在paintGL中)
```

### 结果
- glDrawArrays成功执行40000次顶点绘制
- 但由于顶点属性未绑定，GPU无法访问顶点数据
- 结果是黑屏

## 💡 推荐的解决方案

### 方案A：使用ModernGL库（推荐）
- ModernGL 直接使用libGL，绕过PyOpenGL的context检查
- 支持Core Profile，api更现代
- 与PyOpenGL兼容

```python
import moderngl
ctx = moderngl.create_standalone_context()
```

### 方案B：手动libGL ctypes调用
- 完全绕过PyOpenGL
- 较复杂，需要手动管理所有GL调用
- 但能完全解决context问题

### 方案C：请求Compatibility Profile
- 在虚拟GPU上可能不支持
- 如果支持，可使用固定函数管线（glVertex等）
- 需要修改GTK/Gdk配置

### 方案D：使用完全不同的方案
- 考虑使用Vispy或其他点云库
- 但这会改变项目结构
